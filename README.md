## ⚡ Performance Benchmarks (Scale Saturation Analysis)

HireFlow is architected to handle extreme concurrency. Through **Node.js Clustering** and **Redis Session Caching**, the system was stress-tested to identify the physical saturation point of the hardware.

### 🚀 Scalability Results
| Throughput Tier | Requests / Window | Avg Latency | Success Rate | Optimization Strategy |
|:--- |:--- |:--- |:--- |:--- |
| **Raw Infrastructure** | **2,200,000+** | < 1 ms | ✅ 100% | Heartbeat baseline (Express/Cluster) |
| **Authorized Flow** | **647,117** | 21.1 ms | ✅ 100% | **Redis Session Caching** |
| **Cold Auth (Bcrypt)**| **15,005** | 9,841 ms | ✅ 100% | CPU-bound Hashing (Security Tax) |

### 📊 Stress Test Lifecycle (Redis-Backed Sessions)
*Load tested with **Autocannon**. Rate Limiter disabled to identify raw hardware capacity.*

| Phase | Connections | Duration | 200 OK (Success) | Avg RPS | P99 Latency |
|:---|:---|:---|:---|:---|:---|
| 1️⃣ Warm up | 20 | 15s | 29,791 | 1,986 | 42 ms |
| 2️⃣ Ramp up | 50 | 30s | 141,683 | 4,723 | 27 ms |
| 3️⃣ Medium load | 100 | 45s | 198,387 | 4,409 | 68 ms |
| 4️⃣ High load | 200 | 60s | 277,256 | 4,622 | 131 ms |
| **TOTAL** | -- | **2.5 min** | **647,117** | **4,314** | -- |

> **Engineering Insight:** By implementing **Amortized Latency**, the system pays the "Bcrypt Tax" (500ms math) only during the initial handshake. Subsequent requests bypass the CPU bottleneck via the **Redis Cache**, resulting in a **43x increase** in authenticated throughput.
